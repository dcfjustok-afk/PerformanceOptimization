React 性能优化
    主要分 六个方向：
        1.渲染优化
        2.Diff 与虚拟DOM 优化
        3.组件结构优化
        4.异步与懒加载优化
        5.列表渲染优化
        6.构建与资源层优化     

个人总结：
    
level 1:
    1、React内部渲染优化， 不造成 没必要的渲染， 使用
    1.1使用React.Memo() 去 缓存组件 ， 当组件的props没有发生变化时，不会重新缓存组件
    1.2 使用useCallback  去缓存函数， 当组件重新渲染会创建新的函数，因此会造成 子组件的重新渲染，通过添加依赖项，当依赖项不发生变化时，父组件就算重新渲染了也不会 创建新的函数，子组件不会重新渲染
    1.3 使用useMemo() 去缓存函数计算的记过，当函数需要长时间执行时， 使用该方法，添加依赖项，当依赖项未发生变化时，该函数不会重新执行

Level 2:
    2.组件结构和数据流优化
    2.1 将组件拆分，因为父组件更新会导致所有的子组件更新（重新渲染），将父组件中拆封为若干个 功能单一的子组件，当父组件重新渲染时，只重新渲染那些 关联到的子组件（本质上还是使用Memo去实现）
    2.2 状态提升和状态局部化 ， 状态提升 能让兄弟组件之间能够通信，但是当子组件改变了数据，会导致父组件更新，进而会导致所有的子组件更新， 因此需要状态局部化，也就是说state 尽可能在最小的子组件中，在结合React.memo 使用 ， 当state改变后，只有props内有该state的组件才会发生重新渲染
    2.3  使用Context来优化数据流 ， 对于 组件和孙子组件，如果 使用 props传递，则会导致 当state发生变化时， 这 一连串的 组件都会重新渲染（孙、父、子），使用Context能够使得当 state发生改变时， 只有消费了该state的 组件才会重新渲染，没有消费该state的组件就不会渲染

Level 3:
    3、异步加载和虚拟化
    3.1 使用懒加载的方式动态的导入组件或者代码，使用 React.lazy() 搭配占位符 suspense去实现但首次 加载该页面是不会全量的对所用得到的组件进行加载
    3.2 列表虚拟化， 当渲染大量的列表时， 虚拟化 可以 只渲染其中窗口中可视区域的DOM，超出可视范围的不渲染

Level 4 :
    4、构建层与资源层 优化
    4.1 使用 Tree shake ，当项目在打包的时候，移除未使用到的代码，减小bundle的体积
    4.2 代码分割， 动态的加载代码，减少初始包的体积 ，也就是使用懒加载对代码进行异步加载
    4.3 按需引入 组件模块 ， Ant-Design 等组件库在使用时会全量引入 组件 ， 非常耗能， 应该只引入该文件所需的组件即可，可以使用babel-plugin-import 来自动的按需引入

上面是构建层的优化，下面是资源层的优化
    4.4 使用 浏览器的缓存机制， 对静态组件进行在浏览器本地缓存， 设置文件的hash ，当hash发生改变时 再请求 服务器得到所需要的资源（这一块也就是浏览器 缓存的知识 ， 强缓存和协商缓存）
    4.5 对图片进行压缩， 图片是最影响性能的因素之一，多页面中的图片进行压缩可以减少延迟
    4.6 使用CDN分发机制，将静态的组件部署在CDN服务器上，当用户请求资源时，就近使用CDN节点的资源
    4.7 对要发送的 资源进行 压缩， 压缩要传输的资源的体积，资源到达浏览器后自动的解压，减少 延迟
    4.8 使用 服务器渲染， 服务器将渲染好的网页作为资源 发送给客户端 ，客户端 通过 JavaScript渲染 成交互界面， 这种方式 能够 减少 客户端渲染的时间， 提高 首屏加载速度
